🎯 نظام إدارة رحلة توصيل (Delivery Lifecycle System)
يشبه ما يحصل في تطبيقات الطلبات، لكن مع تعقيد كافي يدرّبك على OOP + Patterns معًا.

🧠 الصورة الكبيرة

أي طلب يمر عبر رحلة (Workflow):

إنشاء الطلب
الدفع
التحقق (Fraud/Stock)
تجهيز الطلب
إسناد مندوب
الشحن
التسليم

🔑 هذه الخوارزمية ثابتة → Template Method

🔄 أين الاختلاف؟
الدفع: كاش / بطاقة / محفظة → Strategy
نوع الطلب: طعام / متجر / دواء → Inheritance + Polymorphism
التحقق: قواعد مختلفة حسب النوع → Specification
الشحن: عادي / سريع → Strategy
الحالة: تتغير أثناء الرحلة → State
إشعارات (SMS/Push) → Observer (خفيف)
